What · 先搞懂01

它做的事，就是一個 PM 帶團隊

動態工作流程（Dynamic Workflows）是 Claude Code 的新功能。簡單說：你給一個任務，Claude 自己分成好幾步，叫來一群 AI 同時做、還會互相檢查，最後把整理好的成果交給你。

如果你當過產品經理，這個功能會非常好懂。你給它一個大任務，它做的每一步，幾乎都對得上 PM 的工作。

PM 的工作 vs 動態工作流程

產品經理 PM動態工作流程
把大專案拆成小任務→自動拆解任務
分派團隊，同時動工→派出上百個 AI 平行執行
驗收成果、抓出問題→派另一批 AI 互相挑錯
盯進度，專案不偏離→進度自動存檔、斷點接續
整合成能交付的成果→交你一份整理好的結果
差別只在一件事：以前你要當 PM，還得自己當底下所有工程師。現在你只出需求，它連團隊都自帶。
Case · 真實案例02

我丟一句需求，它派出一整隊 AI 幫我做完整企劃

這不是官方範例，是我自己跑的。我把一個會影響整季的大任務丟給 Claude 跑工作流程，它做的每一步，剛好把上面那張對照表演了一遍。

1它先像 PM 一樣，把整條流程攤開對齊

我只給了一段話，它沒有急著動工，而是先把整件事拆成一條清楚的流程攤給我看、跟我對齊方向，還列出幾個會影響範圍的關鍵問題，要我先確認才往下跑。

2確認後，派出一整隊 AI 分頭平行作業

它把任務拆成好幾個階段，每個階段派不同的 AI 同時做，還能各自看到用了多少 token、花了多少時間。這就是「一個指令，調動一個團隊」。

3十幾分鐘後，交回一份能直接用的成果

最後它整合成一份完整企劃：核心策略、對標分析、定價方案、5 支 Reels 腳本、自動化漏斗、執行時程表。我要做的只是審稿，不是從零生出來。

How · 開始用03

三種啟動方式，由淺入深

付費方案都能用這功能。Max／Team 預設就開著，不用設定；Pro 要先到 /config 開一下。確認開好之後，差別只在「怎麼觸發」——建議從第二種先試。

最直覺
方式一
指令裡加「workflow」這個字
想單跑一個任務、不改整個 session 設定時最適合

只要你輸入的句子裡有 workflow 這個字，Claude Code 會把它標起來，然後幫你的任務寫一個工作流程，而不是一輪一輪慢慢做。

跑一個 workflow 幫我把這個客戶
做一份完整廣告企劃：對標分析 → 痛點訊息 → Meta Reels 素材
誤判時：按 alt+w 這次忽略，或在那個字後面按 backspace。
最推薦先試
方式二
用內建的 /deep-research
官方現成的工作流程，最適合第一次試水溫

它會針對一個問題，從好幾個角度同時搜尋、交叉比對來源，最後給你一份標好引用的報告。你能親眼看到平行代理跑起來的畫面，又不會像跑整套企劃那樣燒掉一大包額度——最適合第一次拿來感受用量。

/deep-research 2026 年 Meta Reels 廣告素材的最新趨勢與成效數據？

跑起來後，輸入下面指令就能看進度——每個階段的代理數量、token 用量、經過時間：

/workflows # 用方向鍵選任務，Enter 看細節
全自動 · 最吃額度
方式三
開 ultracode 讓 Claude 自己決定
把運算強度拉到最高，整個 session 自動判斷

開了之後，Claude 會自己判斷每個任務要不要動用工作流程，你不用每次都說。但一個請求可能連續變成好幾個工作流程（理解、修改、驗證各一個），每個任務都更吃 token、更花時間。

/effort ultracode
# 開了之後，大任務它會自己判斷要不要動用工作流程
/effort high # 回到日常工作時切回來
Run · 執行時04

第一次跑，會先跳確認

第一次觸發工作流程時，Claude 不會自己偷跑——它會先把這次要做的事攤給你看，你同意了才開始。

從確認到存檔，這樣跑

顯示即將執行的階段
Claude Code 會列出這次工作流程要跑哪些階段
問你要不要允許
CLI 選「Yes, run it」開始；桌面版會跳卡片，有 Once／Always／Deny，還會提醒 token 用量
背景執行，不卡對話
工作流程在背景跑，你的 session 仍然可以繼續用
跑得好就存起來
在 /workflows 選那次執行、按 s，存成你自己的指令，之後打 /名稱 就能重跑同一套
Cost · 最重要05

給 Max 用戶的省額度提醒

⚠ 它真的很吃用量，先看完這幾點

  • 會用掉比一般對話多很多的 token，而且照常算進你方案的用量和速率限制。跑一個大的，很可能一次吃光 5 小時額度。
  • 不過額度比以前鬆：自 2026/5/6 起 Claude Code 的 5 小時上限已永久加倍，且 Pro／Max 取消了尖峰時段的用量限制（Anthropic 靠 SpaceX 算力擴充）——比過去更燒得起，但仍建議從小任務試起。
  • 先用小任務試。從 /deep-research 丟一個你本來就想查的問題開始，別一開始就跑整套企劃。
  • 可指定較小的模型。描述任務時，請 Claude 對不需要最強模型的階段改用小模型；大任務前先用 /model 確認。
  • 隨時可停。在 /workflows 視圖裡可以停掉執行中的工作流程，已完成的部分不會白做。
FAQ · 動手前先看

動態工作流程，常見問題

要哪種方案才能用？我的方案可以嗎？+
不用挑。所有付費方案都能用，差別只在開啟方式：Max／Team 是預設開好的，最省事；Pro 方案要自己到 /config 把「Dynamic workflows」那一列打開；Enterprise 則要管理員開。另外先把 Claude Code 更新到 v2.1.154 以上，舊版本根本看不到這個功能。
我在 claude.ai 對話框或 Cowork 裡也能用嗎？+
不行。動態工作流程是 Claude Code 專屬（CLI、桌面版、VS Code 擴充）。claude.ai 對話和 Cowork 這次拿到的是另一個新功能「effort 控制」——讓你調整 Claude 對每個回應投入多少思考力，跟動態工作流程是兩回事。想用工作流程，就要在 Claude Code 裡。

限時加碼：Cowork 目前有限時優惠，2026/6/5–7/5 五小時用量加倍、符合資格自動套用（Pro／Max／Team 適用，免費版除外）。
它跟我平常用 Claude Code 有什麼不同？+
平常是「一個 Claude 一輪一輪做」，工作流程是「它自己拆任務、同時派出一群 AI 分頭做、還互相挑錯」。差別在規模和驗證——適合大任務（整套企劃、大量檔案、需要交叉確認的工作），不是拿來回個訊息、改一行字用的。
到底會吃多少額度？我會不會一次就用爆？+
會。它用的 token 遠比一般對話多，而且照常算進你方案的用量。以 Max 的 5 小時窗口來說，跑一個大的很可能一次吃光。所以務必先用小任務試，從 /deep-research 開始，抓到感覺再放大。
有沒有辦法省一點額度？+
有。描述任務時，可以請 Claude 對不需要最強模型的階段改用小模型；大任務前先用 /model 確認目前用哪個模型。另外，跑到一半覺得不對，在 /workflows 視圖裡隨時可以停，已完成的部分不會白做。
跑得好的流程，下次還要重講一遍嗎？+
不用。在 /workflows 選那次執行、按 s 存成你自己的指令，之後打 /名稱 就能重跑同一套流程。像「廣告企劃」這種你會重複做的，存起來一次，以後一個指令搞定。
它會不會自己亂跑、做我沒同意的事？+
不會。第一次觸發時它會先顯示要跑哪些階段、問你要不要允許（桌面版有 Once／Always／Deny 選項，還會提醒 token 用量）。你同意了才開始，全程在背景跑、不卡你的對話。
